Development Methodology

Agile Framework, SCRUM

Sprints run on a repeating cycle: planning every other week, standups on Mondays 3-4 PM and Thursdays 5-6 PM with interim Discord updates, and sprint demo plus retrospective at the end of each sprint. Demos are held with sponsors Janice and Katherine on Tuesdays, and retrospectives capture improvements, action items, and velocity trends for the next sprint.

Core activities

Requirements are translated into user stories in the “As a [user], I want to [action], so that [goal]” format, then formed into a product backlog for sprint selection during planning. Each sprint executes the selected work, monitors blockers, and updates dependencies, culminating in a review/demo and retrospective before refining the backlog for the next sprint.

Tools

Project management and collaboration use Trello, Discord, Google Drive, and Figma for UX design and prototypes. Technical tooling includes GitHub/Actions for version control and CI, Insomnia/Postman for API testing, flutter_test and PyTest for unit tests, plus Flutter, AWS, pgVector, Python, Doxygen, and Terraform for the implementation stack.

Roles

Product Owner is Janice, setting product vision, direction, and accepting/rejecting work at review. Angela serves as Project Manager and Scrum Master, bridging stakeholders and developers, managing and prioritizing the backlog, defining “Done,” coaching on Scrum, and removing impediments. The team includes Developers delivering “Done” increments and tracking metrics, Communication Lead Alec, Back‑end Lead Kalen, Front‑end Lead Aric, Git Czar Alikhan for repo/CI/CD standards, and a Database Administrator for secure, performant data architecture.

Process and Product Metrics

Process Metrics:

Sprint Velocity - Used to track how our development process is progressing based on predicted work to complete vs the actual work completed. We will go over these metrics at the end of our sprints during sprint retrospective meetings.

Product Metrics:

Code Coverage - % of code covered by passing tests and unit tests for the frontend and backend. We will be tracking these via testing frameworks such as Pytest, flutter_test and etc.. depending on which language we will choose to work with.
